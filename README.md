## 小ネタ＆使い方
### github自体の操作
- ファイルクリエイト時に、複数dirを「/」でつなげると一気にdirと中のファイルが生成できる。
- ファイルのrenameは可能。dirは追跡できなくなる関係で、 git mvでコマンドラインからの作業。browserからは無理。
- 新ファイル作って旧dirは削除。の方がマシ？削除は可能。赤字なので驚く。
- 作業したら、とりあえずcommit？履歴が残ってるので、気にせず改造OK
  
### markdownのメモ  
- コメントアウトは htmlのコメントでOK。  難しいこと考えるより早い。
- toc(目次)はない。外部ツールとかで作れと。jekyll側には普通にあるので心配いらん。
- 見出し(下駄マーク)の後ろに空白が必要。動作が安定せん。目次ツールとか。
